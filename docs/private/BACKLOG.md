## Региональный фильтр для Perch v2

У Perch v2 (Google, 14 491 класс — птицы/лягушки/насекомые/млекопитающие) нет официальной meta-модели вида lat/lon/week → priors, как у BirdNET. Сейчас Perch фильтруется только через iNat-наблюдения в радиусе (RegionFilter) — этого недостаточно.

**Идея:** подключить eBird и/или GBIF range maps для построения per-species prior.

Варианты источников:

- **eBird** (только птицы) — Cornell Lab, бесплатный API, region/checklist endpoints. Хорошее покрытие для птиц, ограничено таксономически.
- **GBIF** (occurrence API, все таксоны) — более широкий охват, включая лягушек/насекомых/растения. Менее структурирован под "что встречается в этом регионе сейчас".
- **iNat species_counts** — уже доступен, можно дёргать по радиусу+месяцу и получать список видов с частотами. Нужно учитывать rate-limit.

Открытые вопросы:

- Кэшировать на сетке (gridcell) для оффлайна?
- Как сочетать с уже работающим iNat-фильтром (replacement vs additional weighting)?
- Сезонность: брать ±1 месяц от даты записи?

Не приоритет — текущего iNat-фильтра + BirdNET meta хватает для основного use-case.

## Единый SpeciesDetailsSheet для всех видов

Сейчас поведение при тапе на вид в ReviewScreen различается:

- Незагруженный вид → открывается `SpeciesDetailsSheet` (техническая информация: модели, фрагменты, временные диапазоны)
- Загруженный вид (наблюдение iNat) → inline expand в списке (статус, кол-во ID, комментарии)

**Желаемое поведение:** всегда открывать `SpeciesDetailsSheet`, а для наблюдений дополнительно подгружать iNat-данные прямо в шит — research grade, число подтверждений, комментарии.
