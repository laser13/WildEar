# Contributing

## Workflow Rules

- Do not bypass `.gitignore`.
- Do not use `git add -f` for ignored files.
- Files under `docs/superpowers/` are local working notes, specs, and plans. Keep them out of git.

## Working Style

- Prefer small, focused commits.
- Keep changes aligned with the current task.
- Run the most targeted tests that verify the change before claiming it is done.
